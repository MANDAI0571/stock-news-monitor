# agents/ ― stock-news-monitor 運用エージェント（10人体制・最終版）

スマホから**短い一言**で回すための役割分担。各エージェントは役割を1つに絞り、危険操作は1人に集約する。
基本方針は `TEAM_POLICY.md`（【永久方針】）に従う。目的＝**利益につながる銘柄を早く見つける**。
最重要目標＝**サクシード(9256)型を前日に検出して取り逃さない**。

## 毎日の標準フロー（10人）
1. **6 市場監督** … 今日の地合い（攻め/普通/守る）。守る日は新規を絞る。
2. **1 ハンター** … 地合いを踏まえて候補抽出。
3. **8 チャート博士** … 候補の形を判定（VCP/コイル/ベース等）＋雑魚除外＋ピボット付与。
4. **9 ニュース博士** … 候補に材料を紐づけ（ポジ=後押し/ネガ=除外）。
5. **4 編集者** → **5 門番** … note下書き作成→チェック（NGなら保存停止）。
6. note 下書き完成 → **高重さんがチャート確認 → 投稿（手動）**。
7. **2 ミス検出官** … 取り逃し疑いを調査。
8. **10 バックテスト博士 ＋ 7 学習係** … 結果を T+1/5/20 で検証・記録、週次パターン更新→改善を 1/8/9 へ。

## 開発ルール（新機能の前に必ず）
③職人が commit 前に順に通す: ①`python3 self_test.py` ②`python3 run_screening.py` ③メール生成 ④note下書き生成。
4つ通らないものは commit しない（詳細は `TEAM_POLICY.md`）。

## 安全境界（重要・全員厳守）
- **commit / push**: `03_implementer_git` だけ。他は仕様・指摘までで止める。
- **note公開**: 誰も自動でしない。高重さんが手動で行う。
- **下書き保存の停止権**: `05_quality_gate` が握る。NG なら 04 は保存しない。
- **地合い**: `06_market_manager` の「守る」判定を前段で尊重。全額投入はしない。
- **backtest.py（売買ロジック）**: 原則不可侵。読むのは可、改変は高重さんの明示許可＋03職人のみ（10 バックテスト博士も読むだけ）。
- **公開情報のみ**: 09 ニュース博士は未公開情報・インサイダー禁止。
- **self_test.py**: 実装系は必ず `python3 self_test.py` を通してから commit。
