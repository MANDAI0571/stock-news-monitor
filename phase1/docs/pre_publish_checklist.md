# 本番投稿前 確認チェックリスト

対象: 配信文（X／LINE／メール）とnote本文を、外部に出す前に通す最終ゲート。
作成: 2026-07-20（Fable／Claude）
方針: **機械チェック（compliance_check.py）＋人の目チェック**の二段。機械が exit 0 でも、人の確認欄が全部埋まるまで配信下書きに回さない。

---

## A. 機械チェック（必須・自動）

各配信文とnoteを `compliance_check.py` に通し、**exit 0（合格）**を確認する。

```
python3 phase1/tools/compliance_check.py --input <配信文またはnote>.md --report 校閲レポート_YYYYMMDD.md
echo $?   # 0 なら合格。1 なら要修正（配信中止）
```

機械が見るもの（詳細は compliance_check.py 本体）:
- 断定表現（必ず上がる／確実に儲かる 等）… 検出で不合格
- 推奨・投資助言（買い推奨／買うべき／全力買い 等）… 検出で不合格
- 利益保証・煽り（元本保証／リスクなし／テンバガー確定 等）… 検出で不合格
- 禁止トークン（未取得／OpenWork／残業／有給／nan／None／null／inf）… 検出で不合格
- 未置換プレースホルダ `{{...}}` … 検出で不合格
- 免責文の有無 … 無ければ不合格

exit 1 が出たら、レポートの指摘箇所を直し、再度 exit 0 になるまで配信しない。

---

## B. 人の目チェック（必須・手動）

機械が合格しても、以下を1つずつ確認する。すべて ✅ になってから配信下書きへ。

データの正しさ
- [ ] 数値・銘柄は**その日のGmailスクリーニング実データ**由来か（前日以前の使い回しでないか）
- [ ] 銘柄コード・銘柄名・価格の**転記ミス**がないか（生成元と1件突合）
- [ ] 取得できなかった項目（例: 売買代金）は**行ごと省略**されているか（0や推定で埋めていない）

配信の作り
- [ ] 登録導線リンクに**UTM**が付いているか（source/medium/campaign/content、規約は phase1_collaboration.md）
- [ ] 銘柄チャートリンク（メール）が正しい銘柄コードを指しているか
- [ ] プレースホルダ `{{...}}` が**1つも残っていない**か（テンプレのまま送らない）
- [ ] 免責文が末尾にあるか（機械チェックと二重確認）

重複・安全
- [ ] 同じ記事をClaude系とCodex系の**両方から配信していない**か（一本化ルール）
- [ ] テスト配信・下書き検証分でないか（実配信のみ外部へ）
- [ ] 実投稿・公開は**高重さんの承認済み**か（半自動フェーズは人が最終投稿）

---

## C. 配信直前の最終確認（1行）

- [ ] 上のA（exit 0）とB（全項目✅）が両方そろった

そろっていれば配信下書きへ。1つでも欠けていれば止める。

---

## 運用メモ

- このチェックは「毎回・全チャネル」に適用する（X/LINE/メール/note）。
- 機械チェックのレポート（校閲レポート_YYYYMMDD.md）は保管し、後から遡れるようにする。
- ルールの正本は `docs/phase1_collaboration.md`。矛盾したら協働仕様書を優先。
