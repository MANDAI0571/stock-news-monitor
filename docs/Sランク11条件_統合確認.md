## 1. 結論（先に）

11条件のうち **実装済み 7、不足/未実装 4**。
不足の核心は **③④（移動平均線の「上向き」判定）・⑨（好決算）・⑩（上方修正）**。
現状の `_rank()` は「スコア合計 S≥85」方式で、**11条件をすべて満たすことをSの必須条件にしていない**。
＝スコアが高ければ、③④⑨⑩が欠けていてもSになり得る。ここが戦略ドキュメントとの最大のズレ。

## 3. Sランク判定ロジックのズレ（重要）

現状 `_rank()`（`scoring.py` L146-153）:

```
score >= 85 → S
score >= 70 → A
score >= 55 → B
else        → 見送り
```

- **問題**: Sは「合計85点」で決まる。③④の上向き、⑨好決算、⑩上方修正が無くても、
  他の加点（高値接近・出来高・テーマ・流動性）だけで85点に届けばSになる。
- **戦略ドキュメントの意図**: Sは「11条件**すべて**を満たす銘柄」＝翌朝の寄り買い候補。
- → **ハードゲート方式（11条件AND）＋その中でスコア順** に変えるのが本来の姿。
  現状はスコア単独方式なので、戦略と一致していない。

唯一の決算ガードは「未確認はSにしない」（L115-117）＝ここは戦略と合致。

## 5. 改善候補（提案のみ・未実施／優先度順）

1. **③④（＋200日）MAの上向き判定を追加**（最小実装）:
   `indicators.py`で `ma25_slope = ma25(today) - ma25(N日前)` を計算し、`>0`で上向き。
   `passes_base_filters`にゲート追加 or scoringの必須条件化。データはすでにある（再取得不要）。
2. **Sを「ハードゲートAND」方式へ**: ①〜⑧⑪を必須ゲート、その中でスコア順にS。
   ⑨⑩は取得でき次第ゲートに昇格、未取得時は「業績未確認」で最大A（決算未確認と同じ安全側）。
3. **⑨好決算・⑩上方修正のデータ源確定**: yfinanceの業績は粗いため、
   カブタン等の業績・修正データ取得を検討（取得不可なら「未確認＝最大A」で正直運用）。
4. **押し目／長期押し目カテゴリの新設**: 25日線タッチ・200日線反発を別ランクで出す。
5. **テーマ辞書の拡張**: データセンター・ロボット・インフラ・国策・電力設備を`detect_theme`に追加。
6. **10〜20銘柄キャップ**: 出力段で `head(20)` 相当の上限を入れる。
7. **赤字/債務超過/仕手株の除外**: 業績・値幅異常の判定を追加（データ源確定が前提）。
8. **トレーリングストップ**: +20%到達後に高値-8% or 25日線割れで手仕舞い、を規律版に追加。
